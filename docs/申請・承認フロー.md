## 登場人物
- 申請者
- 承認者

## 登場機能
- 申請＆承認画面
- 一覧画面
- メール送信サーバ


# フロー
- 申請者が申請＆承認画面で申請データを作成し、一時保存する
- 申請者が申請＆承認画面で申請データを完成させ、申請を実行する
- メール送信サーバが承認者へ、申請データを受け付けた事を通知する
- 承認者が一覧画面で該当データの承認処理ボタンを押下して、申請＆承認画面を表示する
- 承認者が申請＆承認画面で差戻ボタンを押下する
- 申請者がメールサーバから、差戻の通知を受け取る
- 申請者が申請＆承認画面でデータを修正して一時保存する
- 申請者が申請＆承認画面でデータ修正を完了させて再度、申請を実行する
- メール送信サーバが承認者へ、申請データを受け付けた事を通知する
- 承認者が一覧画面で該当データの承認処理ボタンを押下して、申請＆承認画面を表示する
- 承認者が申請＆承認画面で承認ボタンを押下する



@startuml 申請・承認プロセス
actor 申請者
actor 承認者
participant "申請＆承認画面" as Screen
participant "一覧画面" as List
participant "メール送信サーバ" as Mail

== 初回申請 ==
申請者 -> Screen: 申請データを作成し、一時保存
申請者 -> Screen: 申請データを完成させ、申請を実行
Mail -> 承認者: 申請データを受け付けた事を通知

== 承認処理 ==
承認者 -> List: 該当データの承認処理ボタンを押下
List --> 承認者: 申請＆承認画面を表示
承認者 -> Screen: 差戻ボタンを押下

== 差戻対応 ==
Mail -> 申請者: 差戻の通知を受け取る
申請者 -> Screen: データを修正して一時保存
申請者 -> Screen: データ修正を完了させて再度、申請を実行
Mail -> 承認者: 申請データを受け付けた事を通知

== 最終承認 ==
承認者 -> List: 該当データの承認処理ボタンを押下
List --> 承認者: 申請＆承認画面を表示
承認者 -> Screen: 承認ボタンを押下

@enduml
